# Elder-First Church Platform - Admin UI Wireframes (V1)

**Version:** 1.0
**Date:** 2025-11-14
**Target:** Admin Mode (role-gated for Admin, Editor, Submitter)

---

## Design Principles

**Elder-First UX:**
- Large text (min 16px body, 20px headings)
- High contrast (WCAG AA minimum)
- 48px touch targets for buttons
- Clear, jargon-free labels
- Progressive disclosure (hide complexity until needed)

**Accessibility:**
- Keyboard navigation on all screens
- Screen reader friendly
- Skip links on complex pages
- Error messages inline and descriptive

## Component Library Notes

**Reusable Components:**

1. **StatusBadge** - Used for bulletin states, priority levels
   - Draft (gray), Approved (blue), Built (green), Locked (red)
   - Urgent (red), High (orange), Normal (gray)

2. **LargeButton** - Primary action buttons
   - Min 48px height, 16px padding
   - Clear contrast, keyboard accessible

3. **Card** - Container for content sections
   - Border radius 8px, shadow for depth
   - Hover state for interactive cards

4. **EmptyState** - When no content exists
   - Friendly illustration + action button
   - Example: "No bulletins yet. Create your first one!"

5. **LoadingSpinner** - Async operations
   - Large, clear spinner (min 48px)
   - "Loading..." text for screen readers

6. **ErrorMessage** - Inline validation errors
   - Red text, icon, clear instructions
   - Example: "âŒ CCLI number required for songs"

7. **SuccessToast** - Confirmation messages
   - Green banner, auto-dismiss after 5s
   - Example: "âœ… Bulletin locked successfully"
